培养目标

不是“听课式培训”，而是“工程交付式训练”。核心是把能力拆成可检验的里程碑。

  • 能独立做出一个可稳定运行的电赛小作品
    从硬件搭建、程序编写到调试与文档交付。
  • 掌握电赛常见模块与调试套路
    采样、滤波、PID、显示、通信、电源、抗干扰、可靠性。
  • 形成团队协作习惯
    版本管理/提交记录、问题分解、周例会汇报与复盘。

筛选机制（把人筛出来）

以“过程 + 结果”的量化评分为主，减少主观印象。

过程分 40% 阶段测评 40% 协作与复盘 20%
过程分：打卡、提交记录、问题闭环；阶段测评：统一任务验收与压力测试；协作复盘：沟通、分工、文档与复盘质量。
统一题目 统一验收脚本 透明评分表 可追溯提交

训练安排（建议节奏）

可按你们学校实际寒假时长调整。下面是典型的 4 周方案。

W1
基础入门与工具链搭建 开发环境、C语言/模块化、GPIO/定时器/ADC、示波器/逻辑分析基本用法
交付：环境截图+第一个可运行 Demo 交付：调试记录（问题-定位-解决）
W2
传感测量与信号处理 采样、滤波、标定、误差分析；串口/显示；工程结构与参数管理
交付：测量小系统（含标定） 验收：精度/稳定性测试
W3
控制算法与可靠性 PID/限幅/抗积分饱和；电源与噪声；异常处理与容错
交付：闭环控制 Demo 验收：压力测试（长时间运行）
W4
综合小作品 + 结营选拔 按照电赛交付标准：功能、稳定性、指标、文档、答辩演示
交付：作品视频 + 说明书 + 代码仓库 终测：统一评分表打分

任务与考核（可直接贴到 PPT/群公告）

下面是一套“可落地”的任务清单，你们可以按方向增删。每项都对应可验收的交付物。

阶段 任务 验收标准 提交物
W1 环境搭建 + 基础外设 Demo 能在指定板卡跑通 GPIO/定时器/串口；代码结构清晰 代码仓库链接 + 演示视频（30–60s）
W2 传感测量 + 标定 给出标定方法；误差与稳定性曲线；能重复复现 测试数据表 + 简短报告（2–4页）
W3 闭环控制 + 保护逻辑 可调参；无明显震荡；异常输入能降级/保护 代码 + 参数说明 + 压测记录
W4 综合作品（模拟电赛题交付） 功能完整、指标达标、连续运行稳定、答辩表达清楚 作品视频 + 说明书 + 答辩PPT
评分建议（可直接用）
功能完成度 30%｜指标达成 25%｜稳定性与鲁棒性 25%｜代码与文档 10%｜协作与复盘 10%

资源与支持

  • 统一开发板/模块清单
    减少不必要的“器材差异”，提升公平性与复现性。
  • 统一验收脚本与测试工况
    长稳运行、干扰注入、边界条件，避免只在“理想环境”能跑。
  • 每周例会 + 复盘模板
    问题清单、root cause、下周计划、风险点。

参与要求

  • 时间投入
    建议每周 ≥ 10–15 小时，至少能保证两次集中训练。
  • 纪律与产出
    按时提交；不抄袭；可复现实验；能说清楚你做了什么。
  • 沟通协作
    遇到问题先自查与记录，再求助；保持同步与透明。

FAQ

零基础能参加吗？

怎么保证公平？

最终怎么组队？

按方向与能力互补组队；以阶段测评分+稳定性为主。
